## توضیحات
در این پروژه پروتکل جدید Reflex را روی Xray-Core پیاده‌سازی کردیم تا مشکلات و محدودیت‌های پروتکل‌های قبلی مثل VMess و VLESS را کاهش دهیم و ترافیک پراکسی را تا حد ممکن شبیه ترافیک عادی کنیم.  
ابتدا ساختار پایه پکیج Reflex را اضافه کردیم، سپس مرحله‌به‌مرحله handshake، احراز هویت، ایجاد و مدیریت جلسه، رمزنگاری فریم‌ها، و در نهایت مکانیزم fallback به وب‌سرور را طبق مستندات پروژه پیاده‌سازی و با تست‌های واحد و یکپارچگی بررسی کردیم.

## نحوه اجرا
1. پیش‌نیازها را نصب کنید:
   - Go (نسخه 1.21 یا بالاتر)
   - Git
2. ریپو را کلون کنید (اگر قبلاً نگرفتید):
   ```bash
   git clone https://github.com/your-username/reflex-repo.git
   cd reflex-repo/xray-core
   ```
3. باینری Xray را بسازید:
   ```bash
   go build -o xray ./main
   ```
4. در ریشه ریپو، فایل `config.example.json` را به `config.json` کپی و مقادیر لازم (مثل UUID و پورت‌ها) را تنظیم کنید:
   ```bash
   cd ..
   cp config.example.json config.json
   # ویرایش config.json با مقادیر دلخواه
   ```
5. سرور Xray با پروتکل Reflex را اجرا کنید:
   ```bash
   ./xray-core/xray -config config.json
   ```
6. کلاینت سازگار با Reflex را به آدرس و پورت تعریف‌شده در `config.json` متصل کنید و اتصال را تست کنید.

## مشکلات و راه‌حل‌ها
یکی از چالش‌های اصلی، زیاد بودن فایل‌ها و ناآشنایی اولیه با ساختار Xray-Core و خود پروتکل‌های VMess، VLESS و Reflex بود.  
برای حل این موضوع ابتدا زمان قابل‌توجهی را صرف مطالعه تاریخچه و طراحی این پروتکل‌ها کردیم و داکیومنت‌های رسمی و کدهای موجود را به‌دقت خواندیم تا درک خوبی از معماری و جریان داده به‌دست بیاوریم.  
بعد از این مرحله، با تکیه بر توضیحات موجود در مستندات همین ریپو (به‌خصوص مراحل step1 تا step5 و بخش تست‌ها)، پیاده‌سازی را مرحله‌به‌مرحله جلو بردیم و در هر قدم با اجرای تست‌ها و بررسی رفتار برنامه، مشکلات را شناسایی و برطرف کردیم.
