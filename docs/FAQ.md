# سوالات متداول

اینجا جواب سوال‌های رایج رو پیدا می‌کنید.

## سوالات عمومی
### باید همه Step‌ها رو پیاده‌سازی کنم؟

بله، همه Step‌ها (1 تا 5) اجباری هستن. Step 5 شامل قابلیت‌های اجباری (Traffic Morphing) و قابلیت‌های امتیازی (ECH، QUIC، etc.) هست.

### می‌تونم از کدهای آماده استفاده کنم؟

نه. باید خودتون پیاده‌سازی کنید. ولی می‌تونید از مستندات و مثال‌های Xray-Core استفاده کنید.

## مشکلات فنی

### بیلد نمی‌شه

چک کنید:
- Go درست نصب شده (`go version`)
- در پوشه درست هستید (`cd xray-core`)
- import path ها درست هستن
- Xray-Core کلون شده (اگر پوشه `xray-core` وجود نداره، از [setup.md](setup.md) دستورات رو ببینید)

### Xray-Core وجود نداره

اگر بعد از clone کردن ریپو reflex، پوشه `xray-core` رو نمی‌بینید:

```bash
cd reflex
git clone https://github.com/XTLS/Xray-core.git xray-core
```

### Handshake کار نمی‌کنه

چک کنید:
- کلیدها درست مبادله می‌شن
- UUID درست ارسال می‌شه
- timestamp و nonce درست هستن
- لاگ‌ها رو چک کنید

### رمزگشایی fail می‌شه

چک کنید:
- کلید جلسه درست استخراج شده
- nonce درست استفاده می‌شه
- payload کامل خوانده شده
- ترتیب خواندن/نوشتن درست هست

### Fallback کار نمی‌کنه

چک کنید:
- Peek درست کار می‌کنه
- بایت‌های peek شده به وب‌سرور فرستاده می‌شن
- وب‌سرور درست اجرا شده
- پورت fallback درست تنظیم شده

## سوالات درباره نمره

### اگر Step 5 رو کامل نکنم چی؟

Step 5 اجباریه و 25 نمره داره (15 نمره برای قابلیت‌های اجباری + 5 نمره امتیازی). اگه فقط قابلیت‌های اجباری رو پیاده‌سازی کنی، 15 نمره می‌گیری. اگه یکی از قابلیت‌های امتیازی رو هم اضافه کنی، 5 نمره اضافی می‌گیری.

### اگر تست‌ها کامل نباشن چی؟

هر تست 2 نمره داره. اگر 5 تست بزنید، 10 نمره می‌گیرید. اگر کمتر بزنید، نمره کم می‌شه.

### کیفیت کد چقدر مهمه؟

خیلی مهمه. 10 نمره برای کیفیت کد و 10 نمره برای مستندات. پس سعی کنید کد تمیز و خوانا بنویسید.

## سوالات درباره پیاده‌سازی

### باید outbound هم پیاده‌سازی کنم؟

برای نمره کامل، فقط inbound کافیه. ولی اگر outbound هم پیاده‌سازی کنید، امتیاز اضافی می‌گیرید.

### می‌تونم از کتابخانه‌های خارجی استفاده کنم؟

برای رمزنگاری و تبادل کلید، می‌تونید از کتابخانه‌های استاندارد Go استفاده کنید (مثلاً `golang.org/x/crypto`). ولی منطق اصلی پروتکل باید خودتون پیاده‌سازی کنید.

### چطوری بفهمم درست کار می‌کنه؟

تست کنید:
1. با کلاینت وصل بشید
2. داده بفرستید
3. چک کنید که داده‌ها درست منتقل می‌شن
4. Fallback رو با مرورگر تست کنید
