## مرحله 1 پروژه (basic structure)
در این مرحله، یک پروتکل جدید به اسم Reflex به Xray اضافه شد. پوشه‌ها و فایل‌های لازم برای inbound و outbound ساخته شدند و فایل config.proto نوشته شد تا شکل کانفیگ مشخص شود. این فایل کامپایل شد تا کد Go مربوط به آن ساخته شود. بعد، یک Inbound Handler اولیه پیاده‌سازی شد که فعلاً فقط اسکلت کار را دارد و منطق اصلی هنوز داخلش نیست. پروتکل Reflex در Xray ثبت شد تا شناخته شود و در نهایت پروژه بدون خطا build شد. این مرحله فقط برای آماده‌کردن ساختار بود و هنوز وارد منطق Handshake و Session نشده‌ایم.

## مرحله 2 پروژه (handshake)
در این مرحله، منطق handshake ضمنی و احراز هویت برای پروتکل Reflex پیاده‌سازی شد. سرور ابتدا ترافیک ورودی را بررسی می‌کند تا مشخص شود بسته با magic number شروع شده یا شبیه یک HTTP POST معمولی است. در حالت HTTP، داده‌ی handshake از فیلد data به صورت base64 استخراج و decode می‌شود. سپس اطلاعات handshake شامل کلید عمومی موقت، UserID، timestamp و nonce parse می‌شود. سرور با استفاده از X25519 کلید مشترک را محاسبه کرده و از آن یک session key می‌سازد. بعد از آن، کاربر با UUID احراز هویت می‌شود و در صورت نامعتبر بودن، اتصال به fallback فرستاده می‌شود. اگر احراز هویت موفق باشد، سرور پاسخ handshake شامل کلید عمومی خودش و PolicyGrant رمزنگاری‌شده را ارسال می‌کند و در نهایت وارد مرحله Session Handling می‌شود. این مرحله پایه‌ی رمزنگاری، احراز هویت و مخفی‌سازی ترافیک پروتکل را کامل می‌کند.
